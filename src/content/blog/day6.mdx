### 正文開始

如何開始加入一個 Coding 專案。

```
How to get involved in a new project？
 
1. Find a mentor - 尋找一個前輩/導師 
2. Get an overview - 聽前輩介紹 project 的概要
3. Do a small quest - 執行一個小任務
4. Have a don’t-get-stuck-rule - 別自己卡太久
5. Use code review - 跟你的同事一起看 code !
```
以上是我自 2020/4/30 遠端工作的時候，節錄下來的一段話。

```
沒想到這段話一直到現在，意外仍然受用。

而且，直到自己決定開始加入自媒體了，仍然適用。
以下會說明我自己如何利用專案背景，開始自己的自媒體世界 - 30 days 專案。
```
##### 1. 首先建立一個 [Github Repo](https://github.com/tomz12321)，並且開啟 Projects 分頁。

裡面可以長得像這樣：
![image](/static/images/Day6/day6-1.png)

##### 2. 把每天思考出來的新 idea 用卡片的方式，記錄下來。
##### 3. 然後就是專案執行 (Scrum)

> Scrum helps people and teams deliver value incrementally in a collaborative way.
> 
> As an agile framework, Scrum provides just enough structure for people and teams to integrate into how they work, while adding the right practices to optimize for their specific needs.

##### 4. 定時檢視自己的目標 (Milestones)

舉例來說，我自己訂了六個 Iteration (每個 Sprint 2-3 週)，所以當每次 Iteration 跑完的時候，我都可以定時回頭檢視自己做了些什麼。是否有產出所謂的 [MVP](https://en.wikipedia.org/wiki/Minimum_viable_product) 產品。
