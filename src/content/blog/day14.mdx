### 正文開始

在軟體工作室工作，不小心會知道的兩三事。

```
前言，因為長期在海外生活，也一直都在小型的軟體工作室打滾。

不小心就會進去跳進服務雲，一下維護後端，一下維運，一下再跑回前端。

但是好處就是不小心就會知道一下，架站維運的小知識。
```

今天，又再度回到了什麼都沒有的生活。

#### 但我們今天要來聊 Upptime。

是的。我在我的網站上掛了 Upptime。 https://github.com/tomz12321/upptime

#### 大概來講會長這樣：

簡單來說就是如果網站掛掉了，我會馬上收到通知。

然後就可以 24/7 不間斷營運了？

```
雖然說是依靠在網路上的工程師，我也不想在六日工作...
```
於是乎，基本的 backup 備援機制就在六日被啟動了
```
事情是這樣的。星期六的悠閒下午，筆者正在帶著小孩出沒在台北市的榮星花園？

突然接收到手機的震動通知。

實際的 email log 訊息大概是長這樣...
```
![image](/static/images/day14/screenshot-14-2.jpg)
```
看起來是被 github action 自己備援了。
```
什麼!!! 
```
HTTP code:0 
response time 0 ms
```

也就是說，我的 網站 被擊落了!!! =口=!!

但是七分鐘後，自己就因為備援機制給復活了。

#### 來個小廢廢的結論
